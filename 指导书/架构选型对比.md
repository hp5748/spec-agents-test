### 2.1 LangChain vs LlamaIndex vs CrewAI

| 特性 | LangChain | LlamaIndex | CrewAI |
|------|-----------|------------|--------|
| **定位** | 通用Agent框架 | RAG专用 | 多Agent协作 |
| **学习曲线** | 中等 | 简单 | 简单 |
| **灵活性** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **RAG能力** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

| **工具生态** | 丰富 | 中等 | 中等 |

| **社区活跃度** | 极高 | 高 | 中等 |
| **适用场景** | 通用开发 | 知识库问答 | 团队协作模拟 |

### 2.2 深度分析

#### **LangChain** - 行业标准

```python
# 优点
✅ 生态最丰富，社区最活跃
✅ 支持所有主流LLM和向量数据库
✅ LangGraph扩展支持复杂工作流
✅ 文档完善，教程多

# 缺点
❌ API变化快，版本升级成本高
❌ 抽象层次多，调试困难
❌ 性能开销较大

# 最适合
- 企业级Agent开发
- 需要高度定制的场景
- 团队有较强工程能力
```

**LangChain示例代码：**
```python
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain.tools import Tool
from langchain.prompts import ChatPromptTemplate, MessagesPlaceholder

# 定义工具
tools = [
    Tool(
        name="Calculator",
        func=lambda x: eval(x),
        description="用于数学计算"
    )
]

# 创建Agent
llm = ChatOpenAI(model="gpt-4", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有用的助手"),
    ("human", "{input}"),
    MessagesPlaceholder(variable_name="agent_scratchpad"),
])

agent = create_openai_functions_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 执行
result = agent_executor.invoke({"input": "123 * 456等于多少？"})
```

#### **LlamaIndex** - RAG专家

```python
# 优点
✅ RAG场景优化最好
✅ 文档索引和检索能力强
✅ 代码简洁，易于上手
✅ 支持多种数据源

# 缺点
❌ 非RAG场景能力一般
❌ 多Agent支持较弱
❌ 社区相对小

# 最适合
- 知识库问答系统
- 文档检索和摘要
- 企业知识管理
```

**LlamaIndex示例代码：**
```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader

```

#### **CrewAI** - 多Agent协作

```python
# 优点
✅ 多Agent协作最直观
✅ 角色定义清晰
✅ 代码简洁易懂
✅ 支持顺序/并行任务

# 缺点
❌ 生产稳定性待验证
❌ 定制能力有限
❌ 文档不够完善

# 最适合
- 模拟团队协作
- 复杂任务分解
- 原型快速验证
```

**CrewAI示例代码：**
```python
from crewai import Agent, Task, Crew
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4")

# 定义Agent

    role="研究员",

    llm=llm

    role="作家",
    goal="撰写高质量文章",
    backstory="你是一个专业作家",
    llm=llm
)

# 定义任务
research_task = Task(
    description="研究AI发展趋势",
    agent=researcher
)

    description="根据研究结果写一篇文章",
    agent=writer

# 组建团队
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task]

result = crew.kickoff()
```

## 4. 详细技术方案

### 4.1 方案A: LangChain全栈方案

**适用场景**: 企业级Agent应用

```python
# 架构设计
"""
┌─────────────────────────────────────────┐
│            LangChain全栈架构             │
├─────────────────────────────────────────┤
│                                         │
│  用户层                                  │
│  ├─ Streamlit Web UI                    │
│  └─ FastAPI REST API                    │
│                                         │
│  Agent层 (LangChain + LangGraph)        │
│  ├─ Router Agent (意图路由)             │
│  ├─ RAG Agent (知识检索)                │
│  ├─ Tool Agent (工具调用)               │
│  └─ Chat Agent (对话管理)               │
│                                         │
│  基础设施层                              │
│  ├─ LangSmith (可观测性)                │
│  ├─ Chroma/Pinecone (向量存储)          │
│  └─ Redis (缓存/会话)                   │
│                                         │
└─────────────────────────────────────────┘
"""

# 核心代码示例
from langchain_openai import ChatOpenAI
from langchain_community.vectorstores import Chroma
from langchain_community.embeddings import OpenAIEmbeddings
from langchain.tools import Tool
from langchain.agents import AgentExecutor
from langgraph.graph import StateGraph, END

# 1. RAG组件
embeddings = OpenAIEmbeddings()
vectorstore = Chroma(
    embedding_function=embeddings,
    persist_directory="./chroma_db"
)
retriever = vectorstore.as_retriever()

# 2. LLM配置
llm = ChatOpenAI(model="gpt-4", temperature=0)

# 3. 工具定义
tools = [
    Tool(
        name="knowledge_search",
        func=lambda q: retriever.invoke(q),
        description="搜索知识库"
    ),
    Tool(
        name="web_search",
        func=web_search_function,
        description="网络搜索"
    )
]

# 4. LangGraph工作流
def build_agent_graph():
    workflow = StateGraph(AgentState)
    
    # 添加节点
    workflow.add_node("classify", classify_intent)
    workflow.add_node("rag_agent", rag_agent)
    workflow.add_node("tool_agent", tool_agent)
    workflow.add_node("chat_agent", chat_agent)
    
    # 添加边
    workflow.add_edge("classify", "rag_agent")  # 或 tool/chat
    workflow.add_edge("rag_agent", END)
    
    return workflow.compile()

# 5. API服务
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class Query(BaseModel):
    question: str
    user_id: str

@app.post("/chat")
async def chat(query: Query):
    agent_executor = build_agent_graph()
    result = agent_executor.invoke({"input": query.question})
    return {"answer": result["output"]}
```

### 4.2 方案B: LlamaIndex RAG方案

**适用场景**: 知识库问答、文档检索

```python
# 架构设计
"""
┌─────────────────────────────────────────┐
│           LlamaIndex RAG架构            │
├─────────────────────────────────────────┤
│                                         │
│  数据层                                  │
│  ├─ 文档加载器 (PDF/Word/网页)          │
│  ├─ 文本切分器                          │
│  └─ 增强处理器                          │
│                                         │
│  索引层                                  │
│  ├─ 向量索引 (默认)                     │
│  ├─ 树索引 (层次化)                     │
│  └─ 关键词索引 (混合检索)               │
│                                         │
│  查询层                                  │
│  ├─ 查询引擎                            │
│  ├─ 聊天引擎                            │
│  └─ 子问题查询                          │
│                                         │
│  优化层                                  │
│  ├─ 重排序                              │
│  ├─ 响应合成                            │
│  └─ 引用追踪                            │
│                                         │
└─────────────────────────────────────────┘
"""

from llama_index.core import (
    VectorStoreIndex,
    SimpleDirectoryReader,
    ServiceContext,
    StorageContext
)
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.postprocessor.cohere_rerank import CohereRerank

# 1. 配置LLM和Embedding
llm = OpenAI(model="gpt-4", temperature=0)
embed_model = OpenAIEmbedding(model="text-embedding-3-small")

service_context = ServiceContext.from_defaults(
    llm=llm,
    embed_model=embed_model,
    chunk_size=1024
)

# 2. 加载文档
documents = SimpleDirectoryReader(
    "./docs",
    recursive=True,
    required_exts=[".pdf", ".md", ".txt"]
).load_data()

# 3. 创建索引
index = VectorStoreIndex.from_documents(
    documents,
    service_context=service_context,
    show_progress=True
)

# 4. 配置查询引擎
query_engine = index.as_query_engine(
    similarity_top_k=10,
    node_postprocessors=[CohereRerank(top_n=5)]
)

# 5. 聊天引擎（支持多轮对话）
chat_engine = index.as_chat_engine(
    chat_mode="condense_plus_context",
    verbose=True
)

# 6. 使用示例
response = query_engine.query("什么是RAG？")
print(response)
print(response.source_nodes)  # 显示来源
```

### 4.3 方案C: CrewAI多Agent方案

**适用场景**: 复杂任务协作、模拟团队工作

```python
# 架构设计
"""
┌─────────────────────────────────────────┐
│          CrewAI多Agent架构              │
├─────────────────────────────────────────┤
│                                         │
│  协调层                                  │
│  └─ Crew (团队编排)                     │
│                                         │
│  Agent层                                │
│  ├─ Agent 1: 研究员                     │
│  ├─ Agent 2: 分析师                     │
│  ├─ Agent 3: 写手                       │
│  └─ Agent 4: 审核员                     │
│                                         │
│  任务层                                  │
│  ├─ Task 1: 收集信息                    │
│  ├─ Task 2: 分析数据                    │
│  ├─ Task 3: 撰写报告                    │
│  └─ Task 4: 质量检查                    │
│                                         │
│  工具层                                  │
│  ├─ 搜索工具                            │
│  ├─ 文件工具                            │
│  └─ 自定义工具                          │
│                                         │
└─────────────────────────────────────────┘
"""

from crewai import Agent, Task, Crew, Process
from langchain_openai import ChatOpenAI

# 配置LLM
llm = ChatOpenAI(model="gpt-4", temperature=0.7)

# 1. 定义Agent
researcher = Agent(
    role="高级研究员",
    goal="收集全面、准确的信息",
    backstory="""你是一位经验丰富的研究员，擅长从各种来源
    收集和整理信息。你注重细节，确保信息的准确性。""",
    verbose=True,
    allow_delegation=False,
    llm=llm
)

analyst = Agent(
    role="数据分析师",
    goal="深度分析数据，提取洞察",
    backstory="""你是一位资深分析师，擅长从海量信息中
    提取关键洞察，并进行深入分析。""",
    verbose=True,
    allow_delegation=False,
    llm=llm
)

writer = Agent(
    role="内容创作者",
    goal="创作高质量、易读的内容",
    backstory="""你是一位专业作家，擅长将复杂信息
    转化为清晰易懂的内容。""",
    verbose=True,
    allow_delegation=True,  # 可以委托给其他Agent

reviewer = Agent(
    role="质量审核员",
    goal="确保输出质量符合标准",
    backstory="""你是一位严格的审核员，负责检查

    verbose=True,
    allow_delegation=False,
    llm=llm
)

# 2. 定义任务
research_task = Task(
    description="""研究{topic}的最新发展趋势。
    收集以下信息：

    expected_output="一份详细的研究报告，包含数据来源",

)

analysis_task = Task(
    description="""基于研究信息，分析{topic}领域：
    1. SWOT分析
    2. 竞争格局
    3. 关键成功因素
    4. 风险评估
    """,
    expected_output="分析报告，包含图表建议",
    agent=analyst
)

writing_task = Task(
    description="""根据研究和分析结果，撰写一份关于{topic}的综合报告。
    要求：
    1. 结构清晰，逻辑严谨
    2. 数据准确，来源可追溯
    3. 结论明确，建议可行
    """,
    expected_output="一份完整的报告，约2000字",
    agent=writer
)

review_task = Task(
    description="""审核报告质量：
    1. 检查事实准确性
    2. 评估逻辑连贯性

# 3. 组建团队
crew = Crew(
    agents=[researcher, analyst, writer, reviewer],
    tasks=[research_task, analysis_task, writing_task, review_task],

```

### 4.4 方案D: 混合方案（推荐）

**适用场景**: 复杂企业应用

```python
# 架构设计
"""
┌─────────────────────────────────────────────────────────┐
│                  混合架构方案                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  用户接口层                                              │
│  └─ FastAPI + Streamlit                                 │
│                                                         │
│  Agent编排层 (LangGraph)                                │
│  ├─ 意图分类节点                                        │
│  ├─ 路由决策节点                                        │
│  └─ 结果汇总节点                                        │
│                                                         │
│  专业Agent层                                            │
│  ├─ RAG Agent (LlamaIndex) ← 知识问答                   │
│  ├─ Tool Agent (LangChain) ← 工具调用                   │
│  └─ Analysis Agent (CrewAI) ← 复杂分析                  │
│                                                         │
│  基础设施层                                              │
│  ├─ 向量数据库 (Chroma/Pinecone)                        │
│  ├─ 缓存 (Redis)                                        │
│  └─ 监控 (LangSmith)                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
"""

from langgraph.graph import StateGraph, END
from typing import TypedDict, Literal
from llama_index.core import VectorStoreIndex
from crewai import Crew, Agent, Task

# 1. 定义状态
class AgentState(TypedDict):
    query: str
    intent: str
    rag_result: str
    tool_result: str
    crew_result: str
    final_answer: str

# 2. 意图分类节点
def classify_intent(state: AgentState) -> AgentState:
    query = state["query"]
    # 使用LLM分类
    intent = llm_classifier(query)
    state["intent"] = intent
    return state

# 3. RAG节点 (使用LlamaIndex)
def rag_agent(state: AgentState) -> AgentState:
    query = state["query"]
    # LlamaIndex查询
    result = llama_index_engine.query(query)
    state["rag_result"] = str(result)
    return state

# 4. 工具调用节点 (使用LangChain)
def tool_agent(state: AgentState) -> AgentState:
    query = state["query"]
    # LangChain Agent执行
    result = langchain_agent.invoke({"input": query})
    state["tool_result"] = result["output"]
    return state

# 5. 分析节点 (使用CrewAI)
def analysis_agent(state: AgentState) -> AgentState:
    query = state["query"]
    # CrewAI团队协作
    result = analysis_crew.kickoff(inputs={"task": query})
    state["crew_result"] = str(result)
    return state

# 6. 汇总节点
def aggregate_results(state: AgentState) -> AgentState:
    intent = state["intent"]
    
    if intent == "knowledge_query":
        state["final_answer"] = state["rag_result"]
    elif intent == "action_required":
        state["final_answer"] = state["tool_result"]
    elif intent == "complex_analysis":
        state["final_answer"] = state["crew_result"]
    else:
        # 混合结果
        state["final_answer"] = synthesize(
            state["rag_result"],
            state["tool_result"]
        )
    
    return state

# 7. 构建工作流
workflow = StateGraph(AgentState)

# 添加节点
workflow.add_node("classify", classify_intent)
workflow.add_node("rag", rag_agent)
workflow.add_node("tool", tool_agent)
workflow.add_node("crew", analysis_agent)
workflow.add_node("aggregate", aggregate_results)

# 添加条件路由
workflow.add_conditional_edges(
    "classify",
    lambda s: s["intent"],
    {
        "knowledge_query": "rag",
        "action_required": "tool",
        "complex_analysis": "crew",
        "general": "rag"  # 默认走RAG
    }
)

# 所有路径都汇总
workflow.add_edge("rag", "aggregate")
workflow.add_edge("tool", "aggregate")
workflow.add_edge("crew", "aggregate")
workflow.add_edge("aggregate", END)

# 编译
app = workflow.compile()

# 8. 执行
result = app.invoke({"query": "帮我分析AI市场趋势并生成报告"})
print(result["final_answer"])
```
