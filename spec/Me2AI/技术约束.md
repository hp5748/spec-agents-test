# 技术约束

## 技术栈

### 核心框架
- **Agent框架**：LangChain 1.2+（使用 OpenAI 兼容接口）
- **LLM**：DeepSeek-V3.2（通过硅基流动 SiliconFlow API）
- **Web框架**：FastAPI 0.109.0+

### 语言和运行时
- **Python版本**：3.10+
- **包管理**：pip + requirements.txt

### 核心依赖
```
# Agent Framework
langchain==0.2.0
langchain-community==0.2.0
langchain-openai==0.1.8

# Web Framework
fastapi==0.109.0
uvicorn==0.27.0

# Data Validation
pydantic==2.5.0

# Utils
python-dotenv==1.0.0
httpx==0.26.0
openai==1.30.0
```

### 前端技术
- **HTML5 + CSS3**：现代化聊天界面
- **Server-Sent Events**：流式响应长连接
- **Fetch API**：异步HTTP请求
- **CSS动画**：炫酷视觉效果

## 架构决策

### 整体架构
- **简化版单文件结构**：核心逻辑集中在少量文件中
- **模块化设计**：配置、Agent、工具、记忆、提示词分离
- **API服务**：FastAPI提供RESTful接口 + SSE流式接口
- **前端界面**：单页HTML，无框架依赖

### 目录结构
```
.
├── spec/                    # 规范文档
│   ├── Me2AI/              # 人类到AI的沟通文档
│   │   ├── 需求描述.md
│   │   ├── 技术约束.md
│   │   └── 任务规划.md

│   ├── main.py            # FastAPI入口 + 路由定义
│   ├── config.py          # 配置管理
│   ├── agent.py           # Agent核心逻辑（含流式）
│   ├── tools.py           # 工具定义（订单/商品/工单）
```

### Agent设计模式
- **路由Agent模式**：主Agent负责意图识别和路由
- **工具调用模式**：通过工具实现具体功能
- **流式响应**：支持SSE长连接实时输出

### API设计
- **RESTful API**：标准HTTP接口
- **SSE流式接口**：Server-Sent Events实现长连接
- **CORS支持**：跨域资源共享

## 环境配置

### 必需环境变量
```bash
# 硅基流动 API 配置
SILICONFLOW_API_KEY=your_api_key_here
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
SILICONFLOW_MODEL=deepseek-ai/DeepSeek-V3.2

# 服务配置
HOST=0.0.0.0
PORT=8000
LOG_LEVEL=INFO
```

### 获取API Key
访问 [硅基流动](https://siliconflow.cn/) 注册并创建 API Key

## 非功能性需求

### 性能要求
- API响应时间：< 5秒（含LLM调用）
- 流式首字延迟：< 1秒
- 并发支持：10 QPS（Demo级别）

### 安全要求
- API Key通过环境变量管理
- 不在代码中硬编码敏感信息
- 输入验证防止注入攻击
- CORS配置允许跨域访问

### 可用性要求
- 健康检查接口支持
- 错误日志记录
- 优雅的错误处理和用户提示
- 友好的Web交互界面

## 技术限制

### 必须遵守
1. **LLM选择**：使用硅基流动的 DeepSeek 模型（OpenAI兼容接口）
2. **简化原则**：不过度设计，保持最小可运行
3. **API兼容**：使用 OpenAI 兼容的 API 格式

### 灵活调整
1. 提示词模板可根据效果优化
2. 工具实现可使用模拟数据
3. 前端界面可自由定制
