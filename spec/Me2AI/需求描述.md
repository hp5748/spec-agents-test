# 需求描述

## 项目目标

开发一个**智能客服 Agent Demo**，作为智能体开发项目的最小可运行原型，用于验证核心技术路线和功能可行性。

## 用户需求

### 目标用户
- 电商平台客服系统的开发者和决策者
- 需要自动化客服解决方案的企业

### 核心需求
1. **智能对话**：能够理解用户意图并给出准确回复
2. **多场景支持**：覆盖订单查询、商品咨询、投诉处理等常见场景
3. **知识库检索**：基于企业知识库提供专业回答
4. **对话记忆**：支持多轮对话，记住上下文

### 意图类型

1. **订单查询 (order_query)**：用户询问订单状态、物流信息
2. **商品咨询 (product_consult)**：用户询问商品信息、推荐商品
3. **投诉处理 (complaint)**：用户投诉或反馈问题
4. **通用问答 (general_qa)**：常见问题、服务说明

## API 接口

### POST /api/chat
普通对话接口，返回完整响应。

### POST /api/chat/stream
流式对话接口（SSE长连接），实时返回响应内容。

### GET /api/history/{session_id}
获取对话历史。

### DELETE /api/history/{session_id}
清空对话历史。

### POST /api/session/new
创建新会话。

### GET /health
健康检查。

## 验收标准

1. ✅ 能够正确识别4种意图类型
2. ✅ 能够根据意图路由到正确的处理模块
3. ✅ 支持多轮对话，保持上下文连贯
4. ✅ API接口可正常调用并返回结果
5. ✅ 服务可正常启动和运行
6. ✅ 支持流式响应（SSE长连接）
7. ✅ Web聊天界面可正常使用
