# 代码结构

## 文件职责

### src/main.py
**职责**: FastAPI应用入口，路由定义，API接口实现

**关键组件**:
- `app`: FastAPI应用实例
- `ChatRequest`: 对话请求模型
- `ChatResponse`: 对话响应模型
- `agent_cache`: Agent实例缓存（按session_id）

**接口路由**:
- `GET /` - 根路径信息
- `GET /health` - 健康检查
- `POST /api/chat` - 普通对话
- `POST /api/chat/stream` - 流式对话（SSE）
- `GET /api/history/{session_id}` - 获取历史
- `DELETE /api/history/{session_id}` - 清空历史
- `POST /api/session/new` - 创建会话
- `GET /chat` - 聊天页面
- `GET /docs` - API文档

**注意事项**:
- SSE响应需要设置 `X-Accel-Buffering: no` 禁用nginx缓冲
- 流式响应使用 `StreamingResponse` + generator

---

### src/config.py
**职责**: 配置管理，环境变量读取，常量定义

**配置类**:
- `Config`: 主配置类
  - `API_KEY`: 硅基流动API密钥
  - `BASE_URL`: API基础URL
  - `MODEL_NAME`: 模型名称
  - `TEMPERATURE`: 温度参数
  - `MAX_TOKENS`: 最大token数

**枚举定义**:
- `IntentType`: 意图类型枚举
  - `ORDER_QUERY`: 订单查询
  - `PRODUCT_CONSULT`: 商品咨询
  - `COMPLAINT`: 投诉处理
  - `GENERAL_QA`: 通用问答

**常量**:
- `INTENT_NAMES`: 意图代码到名称的映射

---

### src/agent.py
**职责**: Agent核心逻辑，意图识别，路由分发，响应生成

**类定义**:
- `CustomerServiceAgent`: 智能客服Agent
  - `__init__(session_id)`: 初始化LLM和工具
  - `classify_intent(user_input)`: 意图识别
  - `get_tool_by_intent(intent)`: 获取对应工具
  - `process_with_tool(user_input, intent)`: 工具处理
  - `generate_response(user_input, intent)`: 生成响应
  - `chat(user_input)`: 普通对话入口
  - `chat_stream_simple(user_input)`: 流式对话入口

**LLM配置**:
```python
self.llm = ChatOpenAI(
    model=Config.MODEL_NAME,
    api_key=Config.API_KEY,
    base_url=Config.BASE_URL,
    temperature=Config.TEMPERATURE,
    max_tokens=Config.MAX_TOKENS,
)
```

**流式输出格式**:
```
data: {"type": "intent", "intent": "xxx", "intent_name": "xxx"}

data: {"type": "content", "content": "文本块"}

data: {"type": "done"}
```

---

### src/tools.py
**职责**: 工具定义，模拟业务功能

**工具类**:
- `OrderQueryTool`: 订单查询工具
  - 输入: 8位订单号
  - 输出: 订单状态、物流信息

- `ProductSearchTool`: 商品搜索工具
  - 输入: 搜索关键词
  - 输出: 商品列表（名称、价格、描述）

- `CreateTicketTool`: 工单创建工具
  - 输入: 问题类型、描述
  - 输出: 工单号、状态

**工具注册**:
- `get_tools()`: 返回工具列表
- `TOOL_MAP`: 工具名称到实例的映射

---

### src/memory.py
**职责**: 对话记忆管理

**类定义**:
- `ConversationMemory`: 对话记忆类
  - `add_message(session_id, role, content)`: 添加消息
  - `get_history(session_id)`: 获取历史列表
  - `get_history_text(session_id)`: 获取历史文本
  - `clear_session(session_id)`: 清空会话
  - `clear_all()`: 清空所有

**全局实例**:
- `conversation_memory`: 单例实例

**存储格式**:
```python
{
  "session_id": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ]
}
```

---

### src/prompts.py
**职责**: Prompt模板定义

**提示词模板**:
- `INTENT_CLASSIFICATION_PROMPT`: 意图识别提示词
- `ROUTER_PROMPTS`: 路由提示词字典
  - `order_query`: 订单查询
  - `product_consult`: 商品咨询
  - `complaint`: 投诉处理
  - `general_qa`: 通用问答

**模板变量**:
- `{chat_history}`: 对话历史
- `{user_input}`: 用户输入

---

### static/index.html
**职责**: Web聊天界面

**技术栈**:
- 纯HTML5 + CSS3 + JavaScript
- 无框架依赖
- SSE流式接收

**关键功能**:
- 动态背景动画（CSS @keyframes）
- 玻璃态设计（backdrop-filter: blur）
- 流式响应显示（SSE + ReadableStream）
- 打字光标效果（::after伪元素）
- 自动滚动到底部
- 快捷操作按钮

**SSE处理流程**:
1. `fetch('/api/chat/stream', {...})`
2. `response.body.getReader()`
3. 循环 `reader.read()`
4. 解析 `data: {...}` 格式
5. 根据 `type` 字段处理

## 数据流

```
用户输入
    ↓
main.py (API层)
    ↓
agent.py (Agent层)
    ├── classify_intent() → 意图识别
    ├── get_tool_by_intent() → 获取工具
    ├── process_with_tool() → 工具处理
    │       ↓
    │   tools.py (工具层)
    │       ↓
    │   模拟数据返回
    │       ↓
    └── generate_response() → LLM生成
            ↓
        memory.py (记忆层)
            ↓
        返回响应
```

## 扩展指南

### 添加新意图
1. `config.py`: 添加 `IntentType` 枚举值
2. `prompts.py`: 添加对应的路由提示词
3. `agent.py`: 在 `get_tool_by_intent()` 添加映射
4. `tools.py`: 如需要，添加新工具类

### 添加新工具
1. `tools.py`: 继承 `BaseTool` 创建工具类
2. `tools.py`: 在 `get_tools()` 注册
3. `tools.py`: 在 `TOOL_MAP` 添加映射

### 修改LLM
1. `.env`: 修改 API 配置
2. `config.py`: 更新配置类
3. `agent.py`: LLM初始化参数可能需要调整
